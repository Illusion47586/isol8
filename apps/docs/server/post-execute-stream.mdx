title: "Execute stream"

---

Streams execution events as SSE frames.

<ParamField body="request" type="ExecutionRequest" required>
  Execution request with `code` and `runtime`.
</ParamField>

<ParamField body="options" type="Isol8Options">
  Optional execution options merged over defaults.
</ParamField>

<Note>
  Streaming endpoint uses ephemeral execution lifecycle.
</Note>

<ResponseField name="type" type="string">
  Event kind: `stdout`, `stderr`, `exit`, or `error`.
</ResponseField>

<ResponseField name="data" type="string">
  Event payload content.
</ResponseField>

<RequestExample>
```bash
curl -N -X POST http://localhost:3000/execute/stream \
  -H "Authorization: Bearer $ISOL8_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "request": {
      "runtime": "python",
      "code": "import time\nfor i in range(3):\n print(i)\n time.sleep(1)"
    }
  }'
```
</RequestExample>

<ResponseExample>
```text
data: {"type":"stdout","data":"0\n"}

data: {"type":"stdout","data":"1\n"}

data: {"type":"stdout","data":"2\n"}

data: {"type":"exit","data":"0"}
```
</ResponseExample>

## Error responses

### 429 Too Many Requests

Returned when the server's connection queue is full. The semaphore acquire is attempted before the SSE stream starts, so the client receives a normal JSON error response (not an SSE frame).

```json
{
  "error": "Too many requests — queue is full",
  "queue": { "current": 10, "max": 10, "available": 0, "pending": 50 }
}
```

### 408 Request Timeout

Returned when a queued request exceeds `queue.timeoutMs` waiting for an execution slot.

```json
{
  "error": "Request timed out waiting in queue after 30012ms",
  "queue": { "current": 10, "max": 10, "available": 0, "pending": 4 }
}
```

<Tip>
  Queue errors return standard JSON, not SSE frames, so clients should check the response `Content-Type` header or HTTP status code before attempting to parse SSE data.
</Tip>
