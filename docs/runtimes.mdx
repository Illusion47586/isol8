isol8 supports five runtime environments, each built on a minimal Alpine Linux base.

## Supported Runtimes

| Runtime | Docker Image | Default Extension | Inline Support |
|:---|:---|:---|:---|
| [Python](#python) | `isol8:python` | `.py` | ✅ Yes |
| [Node.js](#node-js) | `isol8:node` | `.mjs`, `.cjs`, `.js` | ✅ Yes |
| [Bun](#bun) | `isol8:bun` | `.ts` | ✅ Yes |
| [Deno](#deno) | `isol8:deno` | `.mts` | ✅ Yes |
| [Bash](#bash) | `isol8:bash` | `.sh` | ✅ Yes |

## Package Management Commands

isol8 handles dependencies differently depending on whether you are installing them at runtime (ephemeral) or baking them into a custom image. See [Packages & Images](/packages) for usage guides.

| Runtime | Runtime Install (`isol8 run --install`) | Build Install (`isol8 setup`) |
|:---|:---|:---|
| **Python** | `pip install --user --break-system-packages ...` | `pip install --no-cache-dir ...` |
| **Node.js** | `npm install --prefix /sandbox ...` | `npm install -g ...` |
| **Bun** | `bun install -g --global-dir=...` | `bun install -g ...` |
| **Deno** | `deno cache ...` | `deno cache ...` |
| **Bash** | `apk add --no-cache ...` | `apk add --no-cache ...` |

## Runtime Details

### Python

Uses the system Python 3 provided by Alpine Linux 3.21.

- **Image:** `isol8:python`
- **Extensions:** `.py`
- **Execution:** Runs via `python3 -c <code>` (inline) or `python3 <file>`.
- **Packages:** Installed to user site-packages (`/sandbox/.local`).

### Node.js

Uses the system Node.js (LTS) provided by Alpine Linux 3.21. Supports both CommonJS and ESM.

- **Image:** `isol8:node`
- **Extensions:**
  - `.cjs`: Forces CommonJS execution.
  - `.mjs`: Forces ESM execution.
  - `.js`: Follows Node.js default resolution (looks for `package.json` or defaults to CJS).
- **Packages:** Installed locally in `/sandbox/node_modules`.

### Bun

Uses the latest Bun version available at image build time.

- **Image:** `isol8:bun`
- **Extensions:** `.ts` (Bun takes precedence over Node for `.ts` files).
- **Execution:** Runs via `bun -e <code>` (inline) or `bun <file>`.
- **Packages:** Installed globally in the container.

### Deno

Uses the official `denoland/deno:alpine` image.

- **Image:** `isol8:deno`
- **Extensions:** `.mts` (Used to avoid conflict with Bun's `.ts` default).
- **Execution:** Runs via `deno run <file>`. Inline code is written to a temporary file by isol8 before execution.
- **Packages:** Deno uses URL imports. The "install" step runs `deno cache` to pre-download dependencies.

### Bash

Uses the system Bash provided by Alpine Linux 3.21.

- **Image:** `isol8:bash`
- **Extensions:** `.sh`
- **Execution:** Runs via `bash -c <code>` (inline) or `bash <file>`.
- **Packages:** Installs system packages via `apk add`.

<Note>
  All runtimes run as a non-root user (`sandbox`, uid 100) inside the container, except for specific setup phases (like package installation in custom images) which may run as root before dropping privileges.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Package installation failed">
    Common reasons for installation failures:
    - **Network restrictions:** If running with `--net none` (default), installation will fail. Use `--net host` or `--net filtered` with appropriate whitelisting.
    - **Alpine compatibility:** Some Python/Node packages rely on `glibc` and may not compile on Alpine Linux (`musl`). Look for pre-compiled wheels or alternative packages.
    - **System dependencies:** You may need to install system libraries via `apk` (using a custom image build) before installing certain language packages.
  </Accordion>

  <Accordion title="Permission denied errors">
    The container file system is read-only by default, except for:
    - `/sandbox`: The working directory (writable).
    - `/tmp`: Temporary files (writable, no execution).

    If your code tries to write elsewhere, it will fail. Use `/sandbox` for all file operations.
  </Accordion>
</AccordionGroup>
