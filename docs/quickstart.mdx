## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) installed and running
- [Bun](https://bun.sh) (for running from source) or Node.js 18+

## Installation

<CodeGroup>
```bash npm
npm install isol8
```

```bash bun
bun add isol8
```
</CodeGroup>

For the CLI, install globally:

<CodeGroup>
```bash npm
npm install -g isol8
```

```bash bun
bun install -g isol8
```
</CodeGroup>

## Build Docker Images

Before running any code, build the runtime images:

```bash
isol8 setup
```

This checks Docker connectivity and builds all base `isol8:<runtime>` images (Python, Node, Bun, Deno, Bash).

## Run Your First Code

<Tabs>
  <Tab title="CLI">
    ```bash
    # Inline Python
    isol8 run -e "print('Hello from isol8!')" --runtime python

    # Run a file (runtime auto-detected from extension)
    isol8 run script.py

    # Node.js with packages
    isol8 run -e "const _ = require('lodash'); console.log(_.chunk([1,2,3,4], 2))" --runtime node --install lodash
    ```
  </Tab>
  <Tab title="TypeScript Library">
    ```typescript
    import { DockerIsol8 } from "isol8";

    const isol8 = new DockerIsol8({ network: "none" });
    await isol8.start();

    const result = await isol8.execute({
      code: 'print("Hello from isol8!")',
      runtime: "python",
    });

    console.log(result.stdout); // "Hello from isol8!"
    console.log(result.exitCode); // 0
    console.log(result.durationMs); // ~80

    await isol8.stop();
    ```
  </Tab>
  <Tab title="HTTP Server">
    Start the server:
    ```bash
    isol8 serve --port 3000 --key my-secret-key
    ```

    Make a request:
    ```bash
    curl -X POST http://localhost:3000/execute \
      -H "Authorization: Bearer my-secret-key" \
      -H "Content-Type: application/json" \
      -d '{
        "request": {
          "code": "print(\"Hello from isol8!\")",
          "runtime": "python"
        }
      }'
    ```
  </Tab>
</Tabs>

## Execution Modes

<AccordionGroup>
  <Accordion title="Ephemeral (Default)">
    Each execution gets a fresh container from the warm pool. The container is cleaned and returned to the pool after execution. No state persists between runs.

    This is the fastest and safest mode â€” ideal for one-off code execution.
  </Accordion>
  <Accordion title="Persistent">
    A single container is kept alive across multiple executions. Files, installed packages, and environment state persist between runs.

    Use this for multi-step workflows where later executions depend on earlier ones.

    ```bash
    # CLI
    isol8 run --persistent -e "echo 'data' > /sandbox/state.txt" --runtime bash
    isol8 run --persistent -e "cat /sandbox/state.txt" --runtime bash
    ```

    ```typescript
    // Library
    const isol8 = new DockerIsol8({ mode: "persistent" });
    ```
  </Accordion>
</AccordionGroup>
